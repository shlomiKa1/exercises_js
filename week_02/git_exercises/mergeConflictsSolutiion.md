# פרונות לתרגילים - Merge Conflicts & Pull Requests

---

## פתרון לתרגיל 1

1. תלוי, אם שונו למשהו אחר אז יהיה CONFLICT אם הוסיפו א ז לא יהיה.
2. יהיה להם conflict
3. למשל אם אני שינתי באותו שורה א ז נוצר CONFLICT אחרת לא נוצר ו GIT יודע למזג בינהם.

---

## פתרון לתרגיל 2

1. סוגי סימנים:

- <<<<<<< HEAD -> זה השינוי הנוכחי
- ======= -> הפרדה בין השינויים
- > > > > > > > <branch name> -> מה שהיה לפני השינוי

2. מעכשיו הגיע 30 -> לפני זה הגיע 60
3. זה טיפול ב- conflict כלומר צריך לשנות ידני את הקובץ ואז לאשר

---

## פתרון לתרגיל 3

1. PR זה לפני מיזוג שכך יוכלו לבדוק לפני המיזוג אם כתבו נכון את הקוד
2. Base Brance זה הענף שבו רוצים למזג אליו את הענף Comper branch
3. זה בדיקה של הקוד לראות אם כתבו אותו נכון ובדרך כלל הראש צוות בודק את זה.
4. צריך לאשר את PR שזה טוב.

---

## פתרון לתרגיל 4

1. עורכים ידני בקובץ ומחליטים איזו שורה משאירים ואיזה לא, ואז עושים git add carts.js
2. git commit -m "Fix conflict carts.js"

---

## פתרון לתרגיל 5

1. לוחתים על PR ואז בתיאור כותבים -> "I finish feature/user-avatar, Please check if it's OK".
2. בדרך כלל זה מהראש צוות אבל זה גם יכול להיות חבר'ה שנמצאים בצוות, זה טוב כך שמישהו אחר מסתכל על הקוד ומגלה טעיות לפני שממזגים ל- main.
3. השלבים שצריך לעשות

- 1. בוחרים ידנית איזו שורה רוצים
- 2. עושים git add
- 3. עשוים git commit -m "Fix conflict at feature/user-avatar"

4. git merge -> git commit
5. git branch -d feature/user-avatar (מחיקת ענף)
