## פתרון תרגיל 2

1. טבלה לתיאור התהליך

   | פעולה                                           | שם הפעולה          |
   | ----------------------------------------------- | ------------------ |
   | ברגע שנוצר הקובץ                                | Working Directoiry |
   | הוספת קובץ לשמירה - git add about.html          | Staging Area       |
   | צילום קובץ עם הודעה - git commit -m "<message>" | Commit History     |

2. git add about.html -> git commit -m "Add: HTML file named about"

3. קובץ שנמצא ב- Staging Area הוא מחכה שיכנסו אותו לשמירה בהיסטוריה
   וקובץ שנמצא ב- Working Directory הקובץ בכלל לא ישמר בשמירת היסטורייה, כלומר מחכה להיות ב- Staging Area.

## פתרון תרגיל 3

```bash
1. git status
2. git log --oneline
3. git diff a1b2c3d e4f5g6h
```

4. הבדלים בין:

- git status -> מציג אילו קבצים מחכים לשמירה ואיזה קבצים בכלל לא עלו וכוי.
- git log -> מציג את כל רשימת הקומיטים שהיו עד כה כולל ID (Commit Hash) של הקומיט

## פתרון תרגיל 4

```text
my-app/
├── index.js
├── package.json
├── node_modules/      (תיקייה גדולה שנוצרת אוטומטית ע"י npm)
├── .env               (קובץ עם סיסמאות והגדרות אישיות)
└── README.md
```

1. קבצים שלא כדאי לעקוב אחריהם:

- תיקייה node_modules -> בתיקייה הזאת מכילה את כל ההתקנות של הפרויקט וזה יכול להגיע בסדר גודל של כמה GBS וזה תופס הרבה מקום.
- .env -> קובץ שך כל הסיסמאות למשל סיסמה לDATABASE אישי שכל מי שרואה את הפרויקט לא צריך לדעת מה הסיסמה.

2. קובץ .gitignore

```.gitignore
node_modules
.env
```

3. כמו שכתבנו קודם בסעיף 1, אנשים שהיו רואים את הפרויקט היו רואים את הסיסמאות האישיים של הכותב הפרויקט כמו סיסמה DATABASE.
