# פתרונות - GitHub Remote

---

## פתרון לתרגיל 1

1. עד עכשיו שמרנו את הנתונים במחשב האישי, GITHUB נותן לנו לשמור את הנתונים בענן וכך אנחנו יכולים לעבוד עם צוותים.
2. main / master
3. כי אם אי פעם אנחנו נרצה לשנות משהו בפרויקט קטן או להרחיב אותו אז נוכל לעקוב אחר שינויים.

---

## פתרון לתרגיל 2

1. git remote add origin <url>
2. git push / git push origin main
3. אז היה שגיאה שלא יודע לאיפה לעלות את זה.

---

## פתרון לתרגיל 3

1. git remote add origin <url>
2. git pull
3. פקודה בסעיף 1 זה עבור התחברות פעם ראשונה לפרויקט - פקודה בסעיף 2 זה עדכון הנתונים שנעשו בפרויקט.

---

## פתרון לתרגיל 4

1. git remote add origin <url> / git clone <url>
2. כך שכל מי שנכנס לזה אם זה ציבורי אז לא צריך להכניס סיסמה.
3. מפתח גישה שמשמש כתחליף לסיסמה למי שנכנס בדרך HTTPS.
   בשיטה התחברות לענן זה תלוי:
   - git remote add origin <url> -> זה עבור פרויקט קיים במחשב שאתה רוצה לחבר אותו לענן.
   - git clone <url> -> זה עבור שכפול פרויקט שקיים בענן שרוצים את זה למחשב.

---

## פתרון לתרגיל 5

1. git clone add team-app <url>
2. git pull
3. git pull
4. git push / git push team-app main (First time)
5. git remote add origin <url> -> git push origin main
